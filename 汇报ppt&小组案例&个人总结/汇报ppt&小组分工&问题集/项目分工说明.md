# 项目分工说明

> 项目名称：从传统机器学习到多模态大模型的手写数学公式识别
> 小组成员：3人

---

## 一、分工总览

| 成员 | 角色 | 核心工作 | 工作量占比 |
|:-----|:-----|:---------|:---------:|
| **成员A（你）** | **项目负责人** | 整体架构设计、全部核心代码开发、LoRA微调、DeepSeek混合方案、对比学习探索、评估分析与报告撰写、最终答辩汇报 | **~60%** |
| 成员B | 传统ML开发 | 传统机器学习路线（数据预处理、HOG+LBP+PCA+SVM、实验分析） | ~25% |
| 成员C | 项目支持 | 参与选题分析、PPT制作与排版、环境配置与测试、实验结果整理、文档维护、对比学习部分工作 | ~15% |

## 二、各组员详细工作内容

### 成员A（负责人）— 核心开发与架构

| 工作模块 | 具体内容 | 对应产出 |
|:---------|:---------|:---------|
| 项目架构设计 | 设计三条技术路线（传统ML/对比学习/多模态大模型），规划项目整体结构 | 项目方案文档、README |
| 多模态大模型流水线 | TrOCR + LoRA 微调（700样本，7分钟GPU训练，20%→80.8%） | `finetune_latexify.py` |
| DeepSeek 语义校正 | 设计"LoRA推理+DeepSeek校正"混合方案（80.8%→85.2%） | `main.py` lora_refine模式 |
| 核心源码开发 | 开发 src/ 全部7个模块（InkML解析/图像渲染/API封装/OCR识别/评估/可视化/笔画格式化） | `src/` 全部文件 |
| 对比学习探索 | 实现对比学习编码器、解码器、检索流水线，评估后放弃 | `contrastive_learning/` |
| 矩阵识别模型 | 训练矩阵专用LoRA模型（rank=16，420样本，45.9%） | `scripts/train_matrix.py` |
| 评估体系设计 | 设计EM/CER/BLEU三指标，实现LaTeX归一化（5类等价差异） | `src/evaluator.py` |
| 双路线对比分析 | 全量921样本评估，16维度系统对比，撰写对比分析报告 | `report.md`、`项目汇报1.md` |
| 最终汇报 | 制作PPT并完成项目答辩 | PPT、答辩讲稿 |

### 成员B — 传统机器学习路线

| 工作模块 | 具体内容 | 对应产出 |
|:---------|:---------|:---------|
| 数据预处理 | InkML解析→48×48渲染→数据增强（旋转/缩放/弹性形变）→Top-N筛选 | `preprocess.py` |
| 特征工程 | HOG方向梯度特征（900维）+ LBP纹理特征（10维）→ PCA降维（303维） | `traditional_model.py` |
| SVM模型训练 | LinearSVC / RBF核对比，RandomizedSearchCV调参，68.73%准确率 | `traditional_model.py` |
| 实验分析 | 14张可视化图（混淆矩阵、HOG可视化、PCA方差曲线等） | `experiments/figures/` |
| 实验报告 | 撰写传统ML路线完整实验报告 | `个人工作报告.md`、`个人工作报告.docx` |
