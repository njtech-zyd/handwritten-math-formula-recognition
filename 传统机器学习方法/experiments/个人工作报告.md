## 一、项目背景与目标

基于 CROHME (Competition on Recognition of Online Handwritten Mathematical Expressions) 数据集，探索传统计算机视觉 + 机器学习方法在手写数学公式识别任务上的性能上限，为后续深度学习方案提供对比基线。

**技术路线**: InkML 解析 → 笔画渲染 → 数据增强 → HOG + LBP 特征提取 → StandardScaler → PCA 降维 → SVM（RandomizedSearchCV 调参）

## 三、预处理与特征工程

### 3.1 预处理 Pipeline

1. **InkML 解析**: 提取笔画轨迹点序列 + truth 标签
2. **笔画渲染**: 以笔画中心为基准自适应缩放+居中，渲染为 **48×48** 灰度图，抗锯齿线条
3. **数据增强**（按类别平衡）:
   - 随机旋转 ±10°
   - 随机缩放 ±15%
   - 随机平移 ±2px
   - 弹性形变（50% 概率）
   - 目标: 每类 ≥60 样本, factor=2x

### 3.2 特征提取

| 特征 | 参数 | 维度 |
|------|------|------|
| HOG | orientations=9, ppc=8×8, cpb=2×2 | 900 |
| LBP | P=8, R=1, uniform pattern | 10 |
| **拼接后** | Z-score 标准化 | **910** |

### 3.3 PCA 降维

| 指标 | 数值 |
|------|------|
| 保留方差比 | 95% |
| 降维后维度 | **303**（压缩至原始的 33.3%） |

**分析**: 48×48 的图片尺寸在保证识别效果的同时，将特征维度从 1,774（64×64 情况下）压缩至 910，PCA 后仅需 303 维即可保留 95% 方差说明 HOG 特征在相邻 cell 间存在大量冗余。

## 四、模型训练与调参

### 4.1 搜索策略

- **RandomizedSearchCV**: n_iter=8, 3-fold StratifiedKFold
- log-uniform 采样 C ∈ [10⁻³, 10³], gamma ∈ [10⁻³, 10¹]
- 对比核函数: **LinearSVC** vs **SVC (RBF)**

### 4.2 实验结果

| 核函数 | 最优参数 | CV Accuracy | 训练耗时 |
|--------|---------|:-----------:|---------|
| **LinearSVC** | C = 0.039 | **66.99%** | 1,973s (32.9 min) |
| RBF | C = 0.356, γ = 0.001 | 66.67% | 56s |

### 4.3 最终模型

| 指标 | 数值 |
|------|------|
| 最佳核函数 | **Linear** |
| 最优 C | 0.039 |
| 3-fold CV (Grid) | **66.99%** |
| 5-fold CV (Final) | **66.99% ± 0.18%** |
| **测试集准确率** | **68.73%** |
| 总运行时间 | 2,043s (约 34 分钟) |

### 5.2 混淆分析

误分类的 Top-3 原因:
1. **结构相似的公式**（如 sin 系列 vs cos 系列，上下标组合相似）
2. **包含多级分数的公式**（笔画密集区域视觉特征趋同）
3. **含指数的公式**（指数位置微小差异导致混淆）

混淆矩阵热力图见 `experiments/figures/10_confusion_heatmap.png`。

## 六、关键发现与分析

### 6.1 LinearSVC 优于 RBF

- 在小样本多类别场景下（30 类 × 200+ 样本/类），**线性核泛化性更好**
- PCA 已使特征空间近线性可分，RBF 核的非线性映射反而引入过拟合风险
- LinearSVC 训练时间远长（1973s vs 56s），因为 OVR 策略需训练 30 个二分类器

### 6.2 PCA 降维效果显著

- 910 → 303 维（压缩 **67%**），仅损失 5% 方差
- 48×48 图片 + ppc=8 是性价比最优配置，比 64×64 节省 50% 计算量而准确率基本持平

### 6.3 数据增强是必要的

不做增强时每类仅 ~95 训练样本，30 类 SVM 难以收敛。增强后每类 ~180 训练样本，CV 稳定在 67%。

### 6.4 传统方法的局限

- **68.73% 准确率**在 30 类任务上——随机猜测基线为 3.3%，说明模型学到了有效特征，但距离实用（>90%）仍有差距
- HOG 特征对笔画顺序不敏感，丢失了**在线手写的时间序列信息**
- 完整 LaTeX 公式作为标签，粒度太粗——实际应用需求**单符号级别识别**

**结论**: 传统 HOG+LBP+PCA+SVM 路径在 CROHME Top-30 公式识别上达到 **68.73%** 测试准确率，验证了传统 CV 方法在数学公式识别上的可行性。LinearSVC（C=0.039）为最优模型。但距离实用水平仍有显著差距，建议下一步引入在线笔画时序特征或迁移至深度学习方法。
